操作系统学习指北：从 CS162 劝退到 NJU 重燃

2026-06-15 22:00·阅读时间：7 分钟

操作系统。三个字，足以让无数计算机系学生闻风丧胆。

它是每一份 CS 培养方案里绕不开的必修课，和数据结构、计组、计网并称"四大天王"。但和其他三门不同，操作系统的痛苦指数似乎高出一个量级——它离应用层太远，离硬件又不够近，卡在一个不上不下的位置，充斥着大量几乎无法上手的抽象理论。

最近我开始系统学习操作系统。和大多数自学者一样，我第一步就是打开 csdiy.wiki，翻到操作系统那一栏。排在最前面、评价最高的，是伯克利的 CS162。

CS162：高山仰止，然后放弃

CS162 在 CS DIY 上的评价极高。伯克利出品，John Kubiatowicz 教授授课，配套实验 Pintos，教材经典，体系完整——一切看起来都很完美。

Kubiatowicz 教授的水平毋庸置疑。他是那种你一开口就能感受到深厚功底的老师，每个概念都能从历史讲到实现，从原理讲到 trade-off。课程内容覆盖面极广：线程与同步、CPU 调度、虚拟内存、页表实现、文件系统、分布式系统中的 RPC 和 Raft 共识算法……甚至作业还提供了 C 和 Rust 两种语言版本，可以说是相当与时俱进了。

但问题是，我真的跟不上。

坚持了大概七八节课之后，我不得不承认一个事实：这门课对我现阶段来说，太难了。

当课程深入到内核态的线程切换、页表的具体实现、以及 Pintos 那些涉及内核代码的编程作业时，我感觉自己像在啃一块铁板。每一口下去牙都在疼，但几乎咬不动。Pintos 是一个精心设计的教育操作系统，理论上你能在其中实现线程调度、用户程序支持、文件系统等核心功能——但前提是你能先消化几千行 C 代码的内核框架，然后在不破坏现有功能的前提下小心翼翼地添加新特性。

我做不到，牡蛎牡蛎。或许是我基础还不够扎实，或许是这种从内核源码出发的自底向上路线不适合我的学习风格，或者说我是太懒了（不会吧）。总之，我被劝退了。

NJU 蒋炎岩：操作系统的另一种打开方式

在 CS DIY 上继续往下翻，我找到了一门难度标记明显更低的课程——南京大学蒋炎岩老师（jyy）的操作系统课。

蒋老师的课和 CS162 几乎是一个镜像反转。

他不在大学教室里对着 PPT 念经，而是直接在 B 站开播。每节课都是一场 vibe coding 的狂欢——终端在他手里溜得飞起，命令行、gdb 调试、汇编分析、内核模块，全在真实环境下手操演示。在国内高校老师普遍"上课念 PPT、下课不见人"的刻板印象里，蒋老师的存在简直像一个异类（善良的）。

更让我惊喜的是他的授课内容。操作系统课能讲什么？进程管理、内存管理、文件系统——这些当然都有。但他还会现场演示如何通过修改游戏内存来制作外挂，在《红色警戒》里直接展示内存读写操作如何转化为屏幕上的作弊效果。这不是哗众取宠，而是把"虚拟内存"、"地址空间"、"进程隔离"这些原本飘在空中的抽象概念，直接具象化为屏幕上能看到的数值变化。听完之后你很难忘记——因为你亲眼看见了。

他还讲过 Claude Code 的源码泄漏事件。不是当成八卦来讲，而是借这个案例讨论 AI 时代对软件工程和计算机教育到底意味着什么。他本人是 vibecoding 的坚定鼓励者，课程实验的设计理念之一就是让学生善用 AI 编程工具——不是一切都丢给 LLM 代劳，而是学会在合适的环节借力，把精力集中在真正需要人类判断力的地方。

这恰好击中了我的需求。我日常在写代码时频繁使用 Claude Code 这样的 SWE Agent 工具，但大多数课程对此要么闭口不提，要么直接禁用——仿佛 AI 辅助编程是什么见不得人的作弊手段。蒋老师的课是第一个让我感受到**"AI 原生"氛围**的传统计算机课程，这让我觉得被理解了，而不是被排斥。

当然，这门课不会带你深入到 Linux 内核的进程调度源码，也不会让你手写一个完整的文件系统实现。如果你想成为内核开发者，NJU 的课可能不够深。但对一个被 CS162 劝退后想要重新建立信心、先搞清楚"操作系统到底在干什么"的学生来说，这种不深入内核、先建立整体认知的路线，恰好是解药。

没有优劣，只有适合

我写这些不是为了比较哪门课更好。

CS162 是好课。Kubiatowicz 教授是好老师。Pintos 是好的教育实验。只是它们不适合现阶段的我（大概永远不会适合我）。一个没有足够 C 语言和系统编程经验的学生，硬啃伯克利的内核级操作系统课，失败几乎是必然的。

NJU 的课也不是"简单版 CS162"。它们只是出发点和侧重点不同：CS162 是自底向上的，从内核机制出发，目标是培养能理解甚至编写操作系统的系统程序员；蒋老师的课是自顶向下的，从应用和编程者的视角出发，目标是培养理解操作系统原理、能在其上高效开发软件的工程师。

两门课都是好课，只是通往同一座山的不同路径。

写在最后

如果你也在学操作系统，我的建议是：先搞清楚自己的目标和现阶段的基础。

如果你是那种喜欢从原理出发、看到源码才安心、不畏惧啃几千行 C 代码的人——CS162 会让你收获巨大，Kubiatowicz 教授不会让你失望。

如果你和我一样，需要先看到"这东西到底能干什么"才能建立起学下去的动力，如果你觉得一上来就钻进内核太劝退——那蒋老师的课可能会让你少走很多弯路，甚至让你重新喜欢上这门学科。

操作系统很硬核，但学习它的方式可以不那么硬核。找到适合自己的那一条路，比硬着头皮走一条公认的"正确路线"重要得多。

2026 年 6 月 15 日，在被 CS162 劝退的一个月后。操作系统还是很难，但我不再害怕它了。
